
# Worlds and Robots — учебный полигон по модульной робототехнике

![CI](https://github.com/bazarbaevsabit/worlds_and_robots/actions/workflows/build.yml/badge.svg)
![License](https://img.shields.io/badge/license-MIT-green)

Проект демонстрирует принцип **модульной композиции** робототехнической системы на
базе ROS2 Jazzy, Gazebo и Docker. В репозитории собраны отдельные пакеты для
описания мира, моделей роботов и управления, которые связываются между собой
исключительно через ROS‑топики.

## 1. Назначение проекта

- Показать, как разбивать робототехническое приложение на независимые ноды,
  запускаемые через launch‑файлы и общающиеся через топики.
- Предоставить студентам готовый полигон с двумя телеуправляемыми роботами
  (мобильная платформа и манипулятор) и окружающим миром.
- Служить шаблоном для выполнения домашних заданий (см. [docs](docs/)).

**Стек:** Ubuntu 24.04, ROS2 Jazzy, Gazebo (Harmonic), Docker.

## 2. Состав и структура

| Элемент | Назначение |
|---------|------------|
| `dockerfile` | Образ контейнера с ROS2 Jazzy Desktop Full и необходимыми пакетами |
| `run.sh` | Скрипт быстрого запуска контейнера с пробросом графики |
| `src/mobile_world` | Пакет, содержащий SDF‑описание симуляционного мира (`world_v1.sdf`) и launch‑файл для его загрузки |
| `src/mobile_description` | Пакет с URDF‑моделью мобильного робота, его датчиками и launch‑файлом |
| `src/arm_description` | Пакет с URDF‑моделью манипулятора и его launch‑файлом |
| `src/arm_teleop` | Собственный пакет телеоперации манипулятора (реализован через `teleop_twist_keyboard` для мобильной платформы и отдельный узел для руки) |
| `docs/` | Гайды по устройству мира, роботов и задания для студентов |

## 3. Архитектура и модульность

Каждый компонент (мир, мобильный робот, манипулятор, телеоп‑узел) реализован в
своём пакете и запускается отдельным launch‑файлом. Связь между ними
осуществляется **только через топики**:

- **Мобильная платформа** получает команды `cmd_vel` из стандартного ROS2‑пакета
  `teleop_twist_keyboard` (не входит в репозиторий, используется штатный из
  Jazzy).
- **Манипулятор** подписывается на топик `/model/simple_knee_arm/joint/knee_joint/cmd_pos`, куда
  собственноручно написанный пакет `arm_teleop` публикует углы шарниров.
- Все описания (URDF/SDF) публикуются через `robot_description` и читаются
  Gazebo и RViz.

> Такой подход позволяет модифицировать или заменять любой компонент,
> не затрагивая остальные, что и является главной демонстрацией модульности.

## 4. Инструкция по запуску

### 4.1. Подготовка Docker‑образа
```bash
docker build -t ros2_full:v1 .
```

### 4.2. Запуск в контейнере
Отредактируйте `run.sh`, указав локальный путь к репозиторию:
```bash
chmod +x run.sh
./run.sh
```
После входа в контейнер автоматически выполнится сборка всех пакетов.

### 4.3. Ручной запуск (при локальной установке ROS2 Jazzy)
```bash
cd src
colcon build
source install/setup.bash
ros2 launch mobile_world world.launch.py
ros2 launch mobile_description mobile.launch.py
ros2 launch arm_description arm.launch.py
ros2 run teleop_twist_keyboard teleop_twist_keyboard         # мобильный
ros2 run arm_teleop arm_teleop_node                           # манипулятор
```

В папке [docs](docs/) собраны пошаговые руководства по изучению SDF‑мира и
URDF‑моделей.
