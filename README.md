# Color Follower — обнаружение и классификация цветных объектов с использованием ROS2 и Gazebo
## 1. Назначение проекта
Проект демонстрирует связку ROS2 Jazzy, Gazebo Harmonic и библиотеки OpenCV для решения задачи распознавания цвета в симулированной среде. Мобильная платформа перемещается вдоль набора цветных кубов, распознаёт цвет каждого куба и выводит его наименование в консоль. Проект предназначен для использования в качестве базового шаблона при изучении компьютерного зрения в ROS2.
## 2. Состав и структура
 `dockerfile` — образ контейнера с ROS2 Jazzy Desktop Full и необходимыми пакетами.
 `run.sh` — скрипт запуска контейнера с пробросом графической подсистемы.
`src/mobile/` — основной ROS2-пакет:
`src/color_follower.cpp` — узел обработки изображений и управления движением.
`launch/launch.py` — запуск Gazebo, мостов, RViz2 и узла распознавания.
`description/robot.urdf` — URDF-модель робота.
`world/test_world.sdf` — файл симуляции с окружающей средой и моделью робота.
`config/view_robot.rviz` — конфигурационный файл RViz2.
## 3. Алгоритм распознавания цвета
Узел `color_follower` подписан на топик `/camera/image_raw`. Обработка каждого кадра включает следующие шаги:
1. Преобразование цветового пространства из BGR в HSV.
2. Бинаризация по насыщенности (S > 80) и яркости (V > 50) для исключения ахроматических областей.
3. Выделение контуров и выбор контура с максимальной площадью (предполагаемый куб).
4. Проверка положения объекта: центр контура должен находиться в пределах ¼ ширины кадра от оптической оси камеры.
5. Вычисление среднего оттенка Hue внутри выбранного контура.
6. Классификация цвета по эмпирическим диапазонам Hue (шкала OpenCV 0–180°):

| Цвет | Диапазон Hue, ° |
|-----------|------------------|
| Красный | 0–10, 170–180 |
| Оранжевый | 10–25 |
| Жёлтый | 25–35 |
| Зелёный | 35–85 |
| Голубой | 85–100 |
| Синий | 100–130 |
| Фиолетовый| 130–160 |

7. При изменении цвета относительно предыдущего кадра выводится информационное сообщение в лог. Команда `cmd_vel` с постоянной линейной скоростью 0.3 м/с подаётся непрерывно, обеспечивая движение платформы.
## 4. Порядок выполнения
### 4.1. Подготовка образа (Docker)
Выполните сборку образа:
```bash
docker  build  -t  ros2_full:v1  .
```
### 4.2. Настройка проброса рабочей директории
В файле `run.sh` укажите корректный локальный путь к склонированному репозиторию (строка `-v ...:/home/work`). При необходимости задайте другие параметры проброса графики.
### 4.3. Запуск в контейнере
```bash
chmod  +x  run.sh
./run.sh
```
После старта автоматически выполнится сборка пакета и запуск всех узлов.
### 4.4. Нативный запуск
При наличии настроенного окружения ROS2 Jazzy сборку и запуск можно выполнить командами:
```bash
cd  src/mobile
colcon  build
source  install/setup.bash
ros2  launch  mobile  launch.py
```

## 5. Ожидаемые результаты
В терминале отображаются названия распознанных цветов по мере прохождения роботом мимо кубов.
Окно Gazebo показывает симулируемую среду и движение платформы.
Окно RViz2 визуализирует модель робота, данные лидара и изображение с камеры.

## 6. Примечания

Для корректной работы графических приложений в контейнере необходимо, чтобы на хосте был запущен X-сервер.
При использовании GPU NVIDIA для ускорения рендеринга в `run.sh` можно добавить `--gpus all` (при установленном `nvidia-container-toolkit`).
Все исходные тексты снабжены подробными комментариями, что позволяет использовать их для самостоятельного изучения.
